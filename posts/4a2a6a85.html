Encryption basics

What is encryption and why is it used?

Encryption scrambles data so that it cannot be understood by anyone that intercepts it unless they know the key that was used to scramble it.

Encryption: The process of turning plaintext into ciphertext.
Decryption: The process of turning ciphertext into plaintext.

Applications of encryption

All encryption works on the basis of turning plaintext into ciphertext

  • Plaintext: data that is readable and understandable.
  • Ciphertext: data has been encrypted.

Caesar cipher, Vernam cipher and transposition ciphers are three common techiniques for encrypting. Here are their introductions:

Methods Introduction
Caesar cipher a substitution cipher where one character of plaintext is substituted for another, which becomes the ciphertext.
Vernam cipher a method of encryption that uses a one-time pad(key)
Transposition cipher a method of encryption where the characters are rearranged to form an anagram.

Key: in cryptography it is the data that is used to encrypt and decrypt the data.
Substitution cipher: a method of encryption where one character is substituted another to create ciphertext.

The Caesar cipher

The Caesar cipher is a substitution cipher that shifts characters in the plaintext by a certain number of positions in the alphabet. For example, if the shift is 3, then the letter A would be encoded as D, the letter B encoded as E, and so on.

Substitution cipher: A substitution cipher is a method of encryption where each letter in the plaintext is replaced by a letter from the ciphertext.

Transposition ciphers

A transposition cipher is a method of encryption where the letters of the plaintext are rearranged in a different order. For example, the letters of the word “hello” could be rearranged to form the word “olleh”. The letters of the word “hello” could also be rearranged to form the word “leloh”.

Rail fence cipher

The rail fence cipher is a transposition cipher that uses a key to encrypt and decrypt a message. The key is used to determine the number of rows in the rail fence. The plaintext is then written in a zigzag pattern across the rows of the rail fence. The ciphertext is then read off in a zigzag pattern across the rows of the rail fence.

Route cipher

a type of transposition cipher that encodes the message by placing it into a grid.

Frequency analysis

Frequency analysis in cryptography it is the study of how often different letters or phrases are used. It is a technique used to break a substitution cipher. It is based on the fact that certain letters appear with different frequencies in the English language. For example, the letter E appears more frequently than the letter Z. This means that if we have a ciphertext that has been encrypted using a substitution cipher, we can use frequency analysis to determine which letters have been substituted for which letters.

Vernam cipher

The Vernam cipher is a type of encryption that uses a key to encrypt and decrypt a message. The key is a random sequence of letters that is the same length as the message. The key is used to determine which letters in the message are encrypted and which letters are not encrypted. The key is also used to determine which letters in the message are decrypted and which letters are not decrypted. The Vernam cioher is an example of a class of encryption techniques known as one-time pad techniques.

  • One-time pad: a key that is only used once to encrypt and decrypt a message and is then discarded.

In theory as long as the pad (or key) is completely random, kept secret and only used once, then it is mathematically impossible to crack the code.

Computational security

computational security recognises the fact that although most encryption can theoretically be cracked, in practice it will be secure enough to withstand most threats.

**The Vernam or one-time pad cipher is the only cipher that is considered to be 100% mathematically secure.**This is because the pad is only used once and then destroyed. This means that code breakers will only ever have one piece of ciphertext to decipher and the key, which was generated randomly, will be generated again for the next piece of plaintext.

  • Computational security: a concept of how secure data encryption is.
  • Computational hardness: the degree of difficulty in cracking a cipher.

How are most codes cracked?

  • Frequency analysis
  • human error (where people do not stick to the system accidentally leave useful information accessible)
  • brute-force attacks.
  • Reverse engineering (where the code is broken by looking at the code itself)
  • Social engineering (where the code is broken by manipulating people into revealing it)
  • Dictionary attacks (where the code is broken by trying common words and phrases)
