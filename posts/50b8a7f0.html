Types of numbers

Two’s complement

A method of working with signed binary values.
Signed binary: binary with a positive or negative sign.

Three steps:

E.X. -75₁₀ to 8-bit binary:

  1. Convert 75₁₀ into binary in 8-bit
  2. Inverse 1s and 0s
  3. Add 1 to the end

Fixed point numbers

Fixed point: where the decimal/binary point is fixed within a number.

Floating point numbers

Floating point: where the decimal/binary point can move within a number.

Fixed VS Floating

Pros of Fixed Pros of Floating
Predictable behavior: Fixed-point arithmetic always yields the same result for the same inputs, regardless of the underlying hardware or implementation. Wide dynamic range: Floating-point arithmetic can represent a much wider range of values than fixed-point arithmetic, making it suitable for applications that require a high degree of numerical precision across a wide range of magnitudes.
Efficiency: Fixed-point arithmetic can be implemented in hardware using simple and inexpensive circuits, making it a cost-effective solution for some applications. Precision accuracy: Floating-point arithmetic can represent real numbers with a high degree of precision, making it useful in applications that require accurate numerical results.
Reduced memory requirements: Fixed-point numbers typically require less storage than their floating-point counterparts, which can be advantageous in memory-constrained applications. Standardization: The IEEE 754 standard for floating-point arithmetic provides a consistent and widely accepted format for representing real numbers, which can be useful for ensuring interoperability between different systems and programming languages.
Precision control: By choosing an appropriate number of fractional bits, fixed-point arithmetic allows for fine-grained control over the precision of calculations, which can be useful in certain applications. Robustness: Floating-point arithmetic can handle a wider range of inputs and computations without overflow or underflow errors than fixed-point arithmetic, making it more robust in certain situations.

Underflow and overflow

Overflow: when a number is too large to be represented with the number of bits allocated.

Underflow: when a number is too small to be represented with the number of bits allocated.

Normalisation and precision

Normalisation: a process for adjusting numbers onto a common scale. In other words, the process of making a number fit the format of a floating point number.
Precision: how accurate a number is.

Rounding errors

Rounding error: the difference between the calculated approximation of a number and its exact mathematical value.

Absolute and relative errors

Absolute error: the difference between the calculated approximation of a number and its exact mathematical value.
Relative error: the absolute error divided by the exact value.
