5.2 Number bases

Created2023-04-21|Updated2023-05-09|Computer Science
|Post View:

Computers process data in the form of zeros and ones which is known as binary data. How binary is used and how it related to other number bases such as decimal and hexadecimal?

  • Number bases: the number of digits available within a particular number system, e.g. base 10 for decimal, base 2 for binary.

The bit

Computers can only process data in binary form(bits), because their processes are digital form(as series of discrete values) and handle pulses of electricity as either on or off, which can be represented as 0s and 1s. 0 and 1 is binary.

Bit: a single binary digit from a binary number - either a zero or a one

All digital data that a computer processes, such as text, images, and sound, are ultimately represented in binary form as a series of 0s and 1s. These binary values are used to represent everything from the individual pixels in an image to the instructions that a CPU uses to carry out computations.

Exam skills:

  1. What is the biggest decimal integer you can represent with n bits?

n bits: 2^n-1

  1. How many different permutations of numbers can you represent with n bits?

n bits: 2^n

The byte

Bits are grouped together to create bytes.

  • A byte is a group of bits, typically 8, used to represent a single character.

  • With one bit we have two permutations: 0 and 1:

  • ∑ = {0, 1}

  • with two bits we have 2²=4 four permutations:

  • ∑² = {00, 01, 10, 11}

  • with three bits we have 2³=8 eight permutations:

  • ∑³ = {000, 001, 010, 011, 100, 101, 110, 111}

  • …

  • A = {0n1n | n ≥ 1}

A single byte can represent 2⁸=256 different values, which is sufficient to represent all the characters in the ASCII character set, as well as a variety of other symbols and control codes. Bytes can also be combined to represent larger values, such as 16-bit, 32-bit, and 64-bit values, which are used for more complex data types, such as integers and floating-point numbers.

Units

Bytes are grouped together to create units: kilobytes, megabytes, gigabytes and terabytes.

A unit is the grouping together of bits or bytes to form larger blocks of measurement.

Number bases

Number bases, also known as numeral systems, are ways of representing the number of digits available within a particular number system, computing uses three main number bases: binary, decimal and hexadecimal.

  • Binary (base-2): e.g. 11010011₂. Uses two digits (0 and 1) to represent numbers. In the binary system, each digit represents a power of 2, and the value of the number is calculated by multiplying each digit by its corresponding power of 2 and summing the results.

  • Decimal (base-10) : e.g. 211₁₀. Uses ten digits (0-9) to represent numbers. In the decimal system, each digit represents a power of 10, and the value of the number is calculated by multiplying each digit by its corresponding power of 10 and summing the results.

  • Hexadecimal or Hex (base-16): e.g. D3₁₆. Uses sixteen digits (0-9 and A-F) to represent numbers. In the hexadecimal system, each digit represents a power of 16, and the value of the number is calculated by multiplying each digit by its corresponding power of 16 and summing the results.

Hexadecimal
Hex is number base 16, using the numbers 0 to 9 and the characters A to F. As it uses a larger number base, fewer digits are needed than binary to represent the same value. For example, D3₁₆ = 211₁₀ = 11010011₂. The implications of this are that it takes less space on screen or print-out, is easier to recognise code patterns, and hence less likely to make mistakes.

Be able to convert between the three number bases

Binary to decimal

  • Write down the binary number.
  • Write down the powers of 2, starting from 2^0 on the right and increasing by 1 for each subsequent power of 2.
  • Multiply each digit of the binary number by its corresponding power of 2.
  • Add up the results of each multiplication to get the decimal equivalent.
  • Example:
  1. Write down the binary number: 101010
  2. Write down the powers of 2: 2^0, 2^1, 2^2, 2^3, 2^4, 2^5 (or 1, 2, 4, 8, 16, 32)
  3. Multiply each digit by its corresponding power of 2:
    1 x 2^5 = 32
    0 x 2^4 = 0
    1 x 2^3 = 8
    0 x 2^2 = 0
    1 x 2^1 = 2
    0 x 2^0 = 0
  4. Add up the results: 32 + 0 + 8 + 0 + 2 + 0 = 42
  5. Therefore, the binary number 101010 is equal to the decimal number 42.

Decimal to binary

  1. Divide the decimal number by 2 and write down the quotient and the remainder.
  2. Divide the quotient from step 1 by 2 and write down the quotient and the remainder.
  3. Continue dividing the quotients by 2 and writing down the remainders until the quotient is 0.
  4. Write the remainders in reverse order to get the binary equivalent.
  • Example:
  1. The decimal is 42
  2. Divide 42 by 2: quotient is 21, remainder is 0
  3. Divide 21 by 2: quotient is 10, remainder is 1
  4. Divide 10 by 2: quotient is 5, remainder is 0
  5. Divide 5 by 2: quotient is 2, remainder is 1
  6. Divide 2 by 2: quotient is 1, remainder is 0
  7. Divide 1 by 2: quotient is 0, remainder is 1
  8. So the remainders, in reverse order, are 1 0 1 0 1 0. Therefore, the decimal number 42 is equal to the binary number 101010.

Decimal to hex

  1. Divide the decimal number by 16 and write down the quotient and the remainder.
  2. Divide the quotient from step 1 by 16 and write down the quotient and the remainder.
  3. Continue dividing the quotients by 16 and writing down the remainders until the quotient is 0.
  4. Convert each remainder into its corresponding hexadecimal digit, using the hex look-up table.
  5. Write the remainders in reverse order to get the hexadecimal equivalent.
  • Example:
  1. The decimal is 1234.
  2. Divide 1234 by 16: quotient is 77, remainder is 2 (which corresponds to the hexadecimal digit 2)
  3. Divide 77 by 16: quotient is 4, remainder is 13 (which corresponds to the hexadecimal digit D)
  4. Divide 4 by 16: quotient is 0, remainder is 4 (which corresponds to the hexadecimal digit 4)
  5. So the remainders, in reverse order, are 42D. Therefore, the decimal number 1234 is equal to the hexadecimal number 42D.

Hex to decimal

  1. Write down the hexadecimal number.
  2. Write down the decimal equivalent of each hexadecimal digit, using the hex look-up table
  3. Multiply each decimal equivalent by 16 raised to the power of its position (counting from right to left, starting at 0 for the rightmost digit).
  4. Add up the results of step 3 to get the decimal equivalent.
  • Example:
  1. The hexadecimal number is 3E7.
  2. The decimal equivalents are 3 for 3, 14 for E, and 7 for 7.
  3. The rightmost digit (7) is in position 0, so its contribution is 7 x 16^0 = 7 x 1 = 7.
  4. The middle digit (E) is in position 1, so its contribution is 14 x 16^1 = 14 x 16 = 224.
  5. The leftmost digit (3) is in position 2, so its contribution is 3 x 16^2 = 3 x 256 = 768.
  6. Add up the contributions from step 3: 7 + 224 + 768 = 999.
