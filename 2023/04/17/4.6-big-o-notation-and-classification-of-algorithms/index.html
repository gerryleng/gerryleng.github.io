4.6 Big o notation and classification of algorithms

|Post View:

Suppose you forget the three-digit password lock, each has 10 numbers, how many times do you have to try to crack the password at worst?

  1. 10C3 = 10! / (3! * (10 - 3)!) = 120
  2. There are 120 combinations, which means that in your worst case, you will have to try 120 times to crack it.

But that’s almost never the case that computer scientists care about. A computer scientist might want to take account of time and space requiments to develop algorithms because many applications are time critical, which means that they have to react virtually instantly. Any delay caused by an inefficient algorithm could prevent the system from working properly. Many applications manipulate very large datasets. Where this is the case, a spacially inefficient algorithm may make the problem intractable.

Functions

Big O notation is used to describe the growth rate of a function. A function is a mathematical concept that relates each element in one set (called the domain) to exactly one element in another set (called the range). In other words, a function assigns an output value for each input value.

  • Domain: all the values that maybe input to a mathematical function.
  • Codomain: all the values that maybe output from a mathematical function.
  • Range: The set of values that are actually produced by the function.

For example, consider the function f(x) = x^2. The domain of this function is all real numbers, because we can plug in any real number for x. The codomain is also all real numbers, because the output of the function can be any real number. However, the range of the function is only the set of non-negative real numbers, because the square of any real number is always non-negative.

Big o notation

Big o notation is also known as asymptotic analysis, it is the method of describing time and space complexity of an algorithm in order to compare the efficiency of different approaches to a problem. With big o notation we express the runtime in terms of how quickly it grows relative to the input size, in the worst-case scenrio: as n gets arbitrarily large.

  • Time complexity: the concept of how much time an algorithm requires.
  • Space complexity: the concept of how much space an algorithm requires.
  • Input size: in big o notation the size of whatever you are asking an algorithm to work with, e.g. data, parameters.

Complexity Description
O(1) Constant time in Big O notation where the time taken to run an algorithm does not vary with the input size
O(logN) Logarithmic time in Big O notation where the time taken to run an algorithm increased or decreases in line with a logarithm.
O(N) Linear time in Big O notation where the time taken to run an algorithm increases in direct proportion with the input size.
O(N²) Polynomial time in Big O notation where the time taken to run the algorithm is a polynomial function of the input size, e.g. the square of the input size.
O(2^N) Expoential time in Big O notation where the time taken to run an algorithm increases as an exponential function of the number of inputs. For example, for each additional input the time taken might double.
  • An algorithm is said to be polynomial time if its running time can be expressed as O(n^k), where n is the input size and k is a constant exponent.

Tractable and intractable problems

In computer science, problems can be classified as either tractable or intractable based on how efficiently they can be solved by an algorithm.

  • Tractable problem: a problem that can be solved in an acceptable amount of time.
  • Intractable problem: a problem that cannot be solved within an acceptable time frame.

A problem is said to be tractable if there exists an algorithm that can solve it in a reasonable amount of time, typically polynomial time in terms of the size of the input. These problems are also known as “easy” problems. For example, sorting a list of integers can be solved in O(n log n) time, where n is the number of elements in the list.

Faced with intractable problems, programmers often produce heuristic algorithms such as Greedy algorithms, A* search…

Heuristic: with algorithms it is a method for producing a ‘rule of thumb’ to produce an acceptable solution to intractable problems.
Because they cannot be solved in a time which is practical with current processing power.

Unsolvable problems

Unsolvable problems: a problem that it has been proved cannot be solved on a computer.

Halting problem: an example of an unsolvable problem where it is impossible to write a program that can work out whether another problem will halt given a particular input.
