4.1 Abstraction and automation

Created2023-04-14|Updated2023-05-09|Computer Science
|Post View:

Logical reasoning

Logical reasoning is the process of using a given set of facts to determine whether new facts are true or false.
More formally it is concerned with the concept of deductive reasoning which originates from the study of maths and philosophy which identifies rules or premises and then applies these to statements to come to a conclusion.
inductive reasoning is another main type of logical reasoning. The difference between them is that deductive reasoning starts with general principles and then applies them to specific cases, while inductive reasoning starts from observations or assumptions, drawing general conclusions from these limited bases of knowledge.

Problem solving

“Problem solving” is what is says on the tin, it is the process of finding a solution to real-life problems. This process involves identifying a problem and then woking out the steps required to solve it. it pervades almost any subject or discipline with analytical content such as mathematics, operations research, computer science and engineering departments.

Abstraction

1. Abstraction is to reduce problems to their essential features. Another way of explaining abstraction is that 2. is the process of finding similarities or common aspects about the problem, while ignoring differences.
There are two types of abstraction you need to know about:

  1. Representational abstraction – the process of removing unnessary details so that only information that is required to solve the problem remains. For example:
  • the use of data structures: a way of organizing and storing data.
  • the use of abstraction layers in computer networks: an abstraction layer is a layer of software that provides a simplified interface to a more complex system.
  1. Abstraction by generalisation/categorisation – The concept of reducing problems by putting similar aspects of a problem into hierarchical categories.
    terms definitions
    Procedural abstraction Procedural abstraction is the concept that all solutions can be broken down into a series of procedures or subroutines.
    Function abstraction breaking down a complex problem into a series of reusable functions.
    Data abstraction hiding how data is represented so that it is easier to build a new kind of data object, e.g. guiding a stack from an array.
    Problem abastraction removing unnecessary details in a problem until the underlying problem is identified to see if this is the same as a problem that has already been solved.
  • Among them, procedural abstraction uses top-down design technique which is defined as related to the modular approach, this starts with the main system at the top and breaks it down into smaller and smaller units a bit like a family tree.

Algorithms

An algorithm is a step-by-step procedure for carrying out a particular task. It usally contains an input, a set of instructions and an output. Programmers use a technique called hand-tracing or dry running to work through the code which is often a nightmare for many test takers, so make sure to practice it.

Information hiding

The process of hiding all details of an object that do not contribute to its essential characteristics. For example, you want to turn on the light, and you just need to know which button to press without knowing how it is implemented.

Decomposition/Composition

  • Decomposition: breaking down a large task into a series of subtasks. Abstraction techniques will be used in order to decompose the system requirements.
  • Composition: building up a whole system from smaller units. The opposite of decomposition. Procedural composition is then the process of creating a working system from the abstraction. This involves:
  • Writing all the procedures and linking them together to create compound procedures
  • Creating data structures and combining them to form compound structures.

Automation

Creating a computer model of a real-life situation and putting it into action. In a subjective sense, automation means saving trouble, because it is the use of software and computer systems to perform tasks that would otherwise require human intervention. Scripting and Machine learning are two main sections.

  • Scripting: writing programs or scripts to automate specific tasks or workflows.
  • Machine learning: is used to automate tasks such as image recognition, neutral language processing…
