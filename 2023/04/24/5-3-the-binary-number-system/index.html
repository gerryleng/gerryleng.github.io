5.3 The binary number system

Created2023-04-24|Updated2023-05-01|A-Level
|Post View:

Adding unsigned binary integers

Unsigned binary: binary that represents positive numbers only.
The basic rules for binary addition are:

  • 0 + 0 = 0 carry 0
  • 0 + 1 = 1 carry 0
  • 1 + 1 = 0 carry 1
  • 1 + 1 + 1 = 1 carry 1
    For example:

Multiplying unsigned binary integers

The basic steps for binary multiplication are:

  • Multiply the first number by each of the digits of the second number in turn starting from the right-hand side.
  • Then do the same for the next digit, shifting your answers to the left as you would in decimal multiplication.
  • Then carry out a binary addition to find the final answer.

    Tip: binary addition and multiplication of positive integers use the same methodology as decimal addition and multiplication.

Types of numbers

Section 5.1 introduces different types of numbers. In this section, we will not only introduce unsigned numbers but also see what methods computers be used to handle real numbers as many computer applications will require very large or very small numbers to be handled to a high degree of accuracy.

Two’s complement

  • Two’s complement is a mathematical method used for representing signed integers in binary form.
  • In this system, the most significant bit (MSB) represents the sign of the number. If the MSB is 0, the number is positive, and if the MSB is 1, the number is negative.
  • To find the two’s complement of a negative number, you invert all the bits (change 0s to 1s and 1s to 0s) and add 1.
  • To convert a positive number to its two’s complement, you simply write its binary representation as is, but add a 0 as the MSB to indicate a positive value.
  • With two’s complement for 8 bits the range is -128 through to 127 and for n bits it would be -2n^(n-1) to 2^(n-1)-1.
  • The benefit of using the two’s complement system is that it allows for easy addition and subtraction of signed numbers, as well as efficient use of computer memory.
Example1

To convert -102 into binary using the two’s complement method:

  1. Convert 102 to binary: 1100110
  2. Invert all bits: 0011001
  3. Add 1: 0011010
    Therefore, -102 in binary using 8 bits and two’s complement representation is 10011010.
Example2

Convert 10011100 into denary using two’s complement:

  1. Check the leftmost bit. Since it is 1, this means the number is negative.
  2. Invert all bits: 01100011
  3. Add 1: 01100100
  4. This gives us the binary representation of -100 in 8 bits.
  5. To convert this into denary, we take the two’s complement binary representation and interpret it as a negative number.
  6. To do this, we first convert the binary number into denary form:
  7. 01100100 = 0x2 + 0x4 + 1x8 + 0x16 + 0x32 + 1x64 = 36
    Therefore, the denary value of 10011100 using two’s complement representation is -36.

Adding and subtracting using two’s complement

The only true arithmetic that the computer can carry out is addition. In order to carry out subtractions, the method used is to convert the number to be subtracted to a negative number, and then to add them together.

Fixed point numbers

Fixed point numbers are a type of numerical representation in computing that store a fixed number of digits (or bits) after the decimal point. This allows for efficient computation of decimal values using integer arithmetic operations.

  • Fixed point: where the decimal/binary point is fixed within a number.
Example1

Convert 3¼ into a binary number

To convert a decimal number to binary in fixed point representation, we first split the decimal number into its integer and fractional parts. In this case, we have 3 and 0.25 as the integer and fractional parts, respectively.

  1. To convert the integer part 3 to binary, we can use the repeated division by 2 method as follows:
  • Divide 3 by 2: 3 ÷ 2 = 1 remainder 1
  • Divide 1 by 2: 1 ÷ 2 = 0 remainder 1
  • The remainders, read from bottom to top, give us the binary representation of the integer part: 3 in binary is 11.
  1. To convert the fractional part 0.25 to binary, we can use the repeated multiplication by 2 method as follows:
  • Multiply 0.25 by 2: 0.25 × 2 = 0.5
  • Record the integer part of the result (0 or 1) as the first binary bit after the decimal point: 0.25 in binary is 0.01.
  • Multiply the fractional part (0.5) by 2: 0.5 × 2 = 1.0
  • Record the integer part of the result (1) as the second binary bit after the decimal point: 0.25 in binary is 0.011.
  • Since the fractional part has now become 0, we stop.

The binary representation of 3.25 in fixed point representation is then the concatenation of the binary representations of the integer and fractional parts: 11.01.

Example2

Convert the binary number 10.11 into its decimal equivalent

Separate the integer and fractional parts of the binary number: 10 is the integer part, and 11 is the fractional part.

  1. Convert the integer part 10 to decimal by multiplying each digit by its corresponding power of 2 and adding the results:
  • 1 * 2^1 + 0 * 2^0 = 2 + 0 = 2
  1. Convert the fractional part 11 to decimal by multiplying each digit by its corresponding power of 2 and adding the results:
  • 1 * 2^-1 + 1 * 2^-2 = 0.5 + 0.25 = 0.75
  1. Add the results of steps 2 and 3 to obtain the final decimal equivalent:
  • 2 + 0.75 = 2.75

Therefore, the binary number 10.11 is equivalent to the decimal number 2.75.

Floating point numbers


Floating point numbers are a type of numerical representation in computing that allow for a wider range of values and greater precision compared to fixed point numbers.

  • Floating point: where the decimal/binary point can move within a number

Fixed point VS Floating point

Advantages of Fixed Point:

  1. Simplicity: Fixed point arithmetic is generally simpler to implement and faster to compute than floating point arithmetic, as it involves only integer operations.
  2. Deterministic: Fixed point arithmetic is deterministic, meaning that the same operations on the same data always produce the same results. This makes it easier to debug and test.
  3. Efficient: Fixed point arithmetic can be more memory efficient than floating point arithmetic, as it requires less storage space for the same range of values.
    Advantages of Floating Point:
  4. Flexibility: Floating point arithmetic is more flexible than fixed point arithmetic, as it can handle a wider range of values and a larger dynamic range.
  5. Precision: Floating point arithmetic can provide much greater precision than fixed point arithmetic, as it can represent both very small and very large numbers with a high degree of accuracy.
  6. Error Handling: Floating point arithmetic includes special values such as NaN (Not a Number) and infinity, which can be used to handle errors and exceptions in numerical computations.

In summary, fixed point arithmetic is generally simpler, faster, and more memory efficient than floating point arithmetic, while floating point arithmetic is more flexible, precise, and includes more error handling capabilities. The choice of which method to use depends on the specific requirements of the application and the available hardware resources.

Normalisation and precision

Normalisation is a technique used to ensure that when numbers are represented they are as precise as possible in relation to how many bits are being used. Precision refers to how accurate the number needs to be in the context that is being used.

  • Normalisation: a process for adjusting number onto a common scale
  • Precision: how accurate a number is.
