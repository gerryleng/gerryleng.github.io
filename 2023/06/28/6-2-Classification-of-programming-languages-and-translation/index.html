6.2 Classification of programming languages and translation

Created2023-06-28|Updated2023-06-28|Computer Science
|Post View:

I. Classification of Programming Languages
A. Low-Level Languages:

Machine Language:
a. Consists of binary code (0s and 1s) understood by the computer directly.
b. Each instruction corresponds to a specific machine operation.
Assembly Language:
a. Uses mnemonics to represent machine instructions.
b. Translated into machine language using an assembler.
B. High-Level Languages:

Designed to be human-readable and easier to understand and write.
Examples: C++, Java, Python.
High-level languages require a translator to convert code into machine language for execution.
C. Procedural Languages:

Emphasize step-by-step procedures or sequences of instructions.
Programs are divided into procedures or functions.
Example: C, Pascal.
D. Object-Oriented Languages:

Based on the concept of objects that encapsulate data and behavior.
Encourages reusability, modularity, and maintainability.
Example: Java, C++.
E. Functional Languages:

Emphasize the evaluation of mathematical functions.
Treat computation as the evaluation of expressions.
Example: Haskell, Lisp.
F. Declarative Languages:

Focus on describing the desired result rather than specifying the algorithm.
Programs describe the logic or constraints of a problem.
Example: Prolog, SQL.
II. Translation of Programming Languages
A. Compilation:

The entire source code is translated into machine code before execution.
Compilation process:
a. Lexical Analysis: Breaks the code into tokens (keywords, identifiers, etc.).
b. Syntax Analysis: Checks the grammar and structure of the code.
c. Semantic Analysis: Verifies the meaning and context of the code.
d. Code Generation: Translates the code into machine language.
e. Optimization: Enhances the efficiency of the generated code.
B. Interpretation:

The source code is translated line by line into machine code during runtime.
Interpretation process:
a. Lexical Analysis: Tokenizes the code.
b. Syntax Analysis: Checks the grammar and structure.
c. Execution: Translates and executes each line one at a time.
C. Just-In-Time (JIT) Compilation:

Combines elements of both compilation and interpretation.
Translates the source code into machine code on the fly during runtime.
Commonly used in languages like Java and C#.
D. Virtual Machines:

An abstraction layer that emulates a computer system.
Intermediary code (bytecode) is executed by the virtual machine.
Provides portability across different hardware platforms.
Examples: Java Virtual Machine (JVM), Common Language Runtime (CLR).
These class notes should provide you with a solid understanding of the classification of programming languages and the translation processes involved. Remember to supplement these notes with further reading, practical examples, and exercises to deepen your knowledge and understanding of the topics.
